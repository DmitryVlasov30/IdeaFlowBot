# Telegram-панель

Этот файл объясняет, как пользоваться новой кнопочной панелью в главном боте.

## Где открыть панель

1. Открой главного бота.
2. Напиши команду `/panel`.
3. Если твой Telegram id есть в `GENERAL_ADMIN` или в списке модераторов, бот покажет кнопки управления.

## Что означают кнопки

### `Обновить входящие`

Эта кнопка запускает импорт из старого collector-слоя.

Что происходит:

- старый саббот уже сохранил сообщение в legacy SQLite;
- importer переносит новые записи в таблицу `submissions`;
- после этого сообщение появляется в новой editorial-панели.

Если бот пишет `Импортировано: 0`, это не ошибка. Это значит, что новых записей с прошлого импорта нет.

### `Поступившие сообщения`

Это очередь новых сообщений, которые еще не прошли первую модерацию.

Проще говоря:

- сюда попадает свежая предложка;
- здесь модератор решает, что делать с сообщением дальше.

Доступные действия:

- `Approve` — создать из сообщения `content_item` и одобрить его;
- `Publish now` — сразу отправить сообщение в publish pipeline;
- `Save as paste` — сохранить сообщение в библиотеку паст;
- `Hold` — временно отложить сообщение;
- `Reject` — отклонить сообщение.

### `Все сообщения`

Это журнал всех `submissions`, которые уже лежат в новой базе.

Здесь видны не только новые сообщения, но и уже обработанные:

- новые;
- отложенные в `hold`;
- превращенные в `content_item`;
- отклоненные;
- отмеченные как пасты.

Если нужно просто просмотреть базу входящих сообщений, открывай именно этот раздел.

### `Черновики на review`

Это не сырые входящие сообщения, а уже собранные черновики постов.

Сюда попадает контент, который уже существует как `content_item`:

- из реального `submission`;
- из пасты;
- из генерации;
- из ручного редакторского создания.

Разница простая:

- `Поступившие сообщения` — сырые входящие;
- `Черновики на review` — почти готовые посты, которые ждут финального решения.

### `Пасты`

Это библиотека паст из таблицы `paste_library`.

Через этот раздел модератор может:

- просматривать все пасты, которые уже лежат в базе;
- добавлять новую пасту вручную;
- убирать пасту из активной работы кнопкой удаления.

Важно:

- кнопка удаления не стирает пасту навсегда;
- сейчас паста переводится в статус `archived`.

Как добавить пасту вручную:

1. Открой `Пасты`.
2. Нажми `Добавить пасту`.
3. Отправь текст одним сообщением.

Если нужен свой заголовок, отправь сообщение в формате:

`Заголовок :: Текст пасты`

### `Каналы и слоты`

Здесь показывается список каналов из editorial-слоя.

После выбора канала можно:

- посмотреть все временные слоты публикации;
- добавить новый слот;
- удалить конкретный слот;
- быстро создать стандартный набор слотов.

Как добавить слот:

1. Открой канал.
2. Нажми `Добавить слот`.
3. Отправь сообщение в одном из форматов:

- `10:00` — слот на все дни;
- `all 10:00` — то же самое;
- `0 09:30` — слот только на понедельник;
- `0,2,4 18:00` — слот на понедельник, среду и пятницу.

Где дни недели:

- `0` — понедельник
- `1` — вторник
- `2` — среда
- `3` — четверг
- `4` — пятница
- `5` — суббота
- `6` — воскресенье

### `Запустить scheduler`

Кнопка вручную запускает планировщик.

Он:

- смотрит на слоты;
- ищет подходящий одобренный контент;
- создает записи на публикацию.

### `Запустить publisher`

Кнопка вручную запускает отправку уже запланированных публикаций в Telegram-каналы.

## Кнопки генерального админа

Эти кнопки доступны только пользователю из `GENERAL_ADMIN`.

### `Сабботы`

Позволяет:

- добавить саббота;
- удалить саббота.

Для добавления отправь строку:

`<api_token> @channel_username`

### `Модераторы`

Позволяет:

- добавить модератора;
- удалить модератора.

Для добавления отправь numeric Telegram user id человека.

## Что теперь показывается по каналу

В карточках сообщений и контента панель старается показывать название или username канала, а не внутренний id editorial-записи.

То есть в интерфейсе модератора ты видишь канал в человекочитаемом виде.

## Рекомендуемый порядок работы

Самый простой сценарий:

1. Нажми `Обновить входящие`.
2. Открой `Поступившие сообщения`.
3. Быстро прими решения по новым сообщениям.
4. При необходимости проверь `Черновики на review`.
5. Если работаешь по слотам, запусти `scheduler`, потом `publisher`.

Если нужно посмотреть историю базы:

1. Открой `Все сообщения` для `submissions`.
2. Открой `Пасты` для библиотеки паст.
